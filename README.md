# PDF RAG Chatbot

Ask questions about your PDFs. Answers are generated by Claude (Opus 4.8) using
**only** retrieved passages, with source + page citations.

## Architecture

```
Ingestion (offline):  PDFs → text → chunks → local embeddings → FAISS index
Query (runtime):      question → embed → search → top-k chunks → Claude → answer + sources
```

Document text is embedded **locally** (`sentence-transformers`); only the
retrieved passages + your question are sent to Claude for answering.

## Setup (VS Code terminal)

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt

Copy-Item .env.example .env      # then paste your ANTHROPIC_API_KEY into .env
```

## Use

1. Drop PDF files into `data/pdfs/`.
2. Build the index:
   ```powershell
   python -m src.ingest
   ```
3. Launch the chat UI:
   ```powershell
   streamlit run app.py
   ```

Re-run step 2 whenever you add or change PDFs.

## Layout

| Path | Responsibility |
|------|----------------|
| `src/config.py` | All tunable settings (chunk size, top-k, model IDs) |
| `src/pdf_loader.py` | PDF → per-page text |
| `src/chunker.py` | Text → overlapping chunks |
| `src/embedder.py` | Local embeddings |
| `src/vector_store.py` | FAISS build / load / search |
| `src/ingest.py` | Offline indexing pipeline |
| `src/rag.py` | Retrieve + Claude answer |
| `app.py` | Streamlit chat UI |

## Tuning

Edit `src/config.py`: raise `TOP_K` for broader context, adjust `CHUNK_SIZE` /
`CHUNK_OVERLAP` for different document types, or swap `CLAUDE_MODEL`.
